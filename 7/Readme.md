# Упаковываем приложение VueJS в Docker контейнер и можем изменять версию NodeJS

Это тестовое приложение, в котором мы добавили возможность изменять версию NodeJS кприложения на jueJS 
В примере используется версия node:24-bookworm-slim

Мне показалось интересным вывести версию NodeJS в index.html

Просто так это не сделать, а вот если добавить переменную окружения в файл package.json на этапе сборки, то это возможно.

## Сначала собираем
```
docker build --target development -t app-dev .
```

## Если вы сначала сделали а потом решили изменить файлы в проекте.

Так как Dockerfile не изменяется, то docker возьмет из кеш и изменения не будут показаны.

```
docker build --no-cache --target development -t app-dev .
```

## Потом запускаем
```
docker run --rm -p 5173:5173 app-dev
```

[!WARNING]
Обратите внимание, что
Vite не поддерживает опцию --root через командную строку напрямую для команды vite build. 
Он ожидает, что вы укажете корень проекта в конфигурационном файле vite.config.js

## Теперь проверим нашу итоговую сборку
```
docker build -t my-nginx-app .
```

## Проверим работу
```
docker run -d -p 8080:80 --name react-app-container my-nginx-app
```

## Так как мы прокидываем порт смотрим тут
http://localhost:8080
